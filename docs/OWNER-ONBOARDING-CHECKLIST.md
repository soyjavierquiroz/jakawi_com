# JAKAWI — Owner Onboarding Checklist v1

Fecha: 2026-07-02
Estado: guia operativa docs-only para private beta controlada.
Uso: aplicar negocio por negocio antes de crear, revisar o publicar un Commercial Space real.

## 1. Objetivo

Este documento sirve para activar negocios reales en private beta de forma asistida, consistente y sin improvisacion. Ordena que pedir al owner, como revisar el Commercial Space, cuando publicar, como operar pago manual y como dar seguimiento semanal.

La private beta de JAKAWI no es self-service masivo. El flujo esperado es onboarding guiado, soporte manual, datos reales separados de demo/QA y cierre comercial por WhatsApp.

## 2. Perfil de negocio recomendado para private beta

Priorizar negocios que:

- Venden por WhatsApp o conversacion.
- Tienen productos o servicios claros.
- Pueden responder rapido a clientes.
- Tienen fotos basicas del producto, servicio o resultado.
- Tienen precios o rangos orientativos.
- Aceptan operacion manual durante beta.
- Estan dispuestos a dar feedback semanal.

Evitar por ahora negocios que:

- Tienen inventario muy complejo.
- Requieren checkout automatico.
- Exigen facturacion automatica inmediata.
- Necesitan multiples sucursales complejas.
- Esperan un marketplace.
- No responden WhatsApp.

## 5. Checklist de productos

Para cada producto:

- [ ] Nombre.
- [ ] Descripcion clara.
- [ ] Precio o rango.
- [ ] Moneda.
- [ ] Categoria.
- [ ] Foto principal.
- [ ] Estado visible/oculto.
- [ ] Destacado si/no.
- [ ] Preguntas frecuentes.
- [ ] Objeciones frecuentes.
- [ ] Variantes/tallas/colores si aplica.
- [ ] Que debe confirmar el vendedor por WhatsApp.

Nota: Seller AI no debe inventar stock, envio, garantia, tallas ni condiciones no cargadas.

## 6. Checklist visual

- [ ] Logo cuadrado.
- [ ] Portada 4:3.
- [ ] Fotos de producto cuadradas o verticales.
- [ ] No usar capturas borrosas.
- [ ] No usar imagenes con demasiado texto.
- [ ] Priorizar producto real.
- [ ] Revisar mobile primero.
- [ ] Confirmar que el CTA no tapa contenido.

## 7. WhatsApp y cierre

- [ ] Numero correcto con pais.
- [ ] Responsable de responder.
- [ ] Horarios.
- [ ] Tiempo esperado de respuesta.
- [ ] Mensaje inicial sugerido.
- [ ] Que informacion debe recibir del cliente.
- [ ] Como marcar lead como ganado/perdido manualmente si aplica.
- [ ] Que hacer si el cliente pregunta precio/envio/disponibilidad.

## 8. Seller AI setup

Solo si el plan lo permite.

- [ ] Activar/desactivar Seller AI.
- [ ] Revisar quick replies.
- [ ] Revisar voz del vendedor.
- [ ] Intro voice note.
- [ ] Guidance voice note.
- [ ] Handoff voice note.
- [ ] Transcripciones obligatorias.
- [ ] Probar widget mobile.
- [ ] Probar handoff a WhatsApp.
- [ ] Confirmar limites por plan.

Reglas:

- Seller AI prepara, WhatsApp cierra.
- Seller AI no promete condiciones no verificadas.
- Seller AI debe reducir incertidumbre, no reemplazar al vendedor.

Plantilla:

```text
Hola, tu espacio comercial en JAKAWI ya esta listo:

- Link publico:
- Usuario/email:
- WhatsApp configurado:
- Productos publicados:
- Plan:
- Que revisar hoy:
- Que hacer cuando llegue un lead:
- Canal de soporte:
- Proxima revision:
```

No incluir password en mensajes.

Si el owner entra en una secuencia de onboarding por email:

- [ ] Confirmar que el email es correcto.
- [ ] Confirmar consentimiento/comunicacion para recibir guias de onboarding.
- [ ] Explicar que los emails automaticos estan en fase de configuracion si aun no estan activos.
- [ ] No incluir passwords, tokens ni datos sensibles en correos o handoffs.

- [ ] Conversion a WhatsApp.
- [ ] Calidad de leads.
- [ ] Tiempo de respuesta del vendedor.
- [ ] Ajustes de productos/fotos/precios.
- [ ] Posible upgrade/downgrade.
- [ ] Testimonial/caso de uso.

## 14. Criterios GO/NO-GO por negocio

GO si:

- WhatsApp correcto.
- Minimo catalogo util.
- Owner entiende operacion.
- Soporte acordado.
- Pago/trial claro.
- No depende de feature inexistente.

NO-GO si:

- No hay WhatsApp operativo.
- Owner no respondera clientes.
- Necesita checkout automatico obligatorio.
- Exige funcionalidades fuera del MVP.
- Datos basicos incompletos.
- Producto/servicio no puede explicarse con claridad.

## 17. Riesgos y limites que se deben comunicar

- Beta privada, no producto final masivo.
- Pagos manuales.
- Sin checkout automatico.
- Sin emails transaccionales por ahora.
- Email onboarding/automation puede existir como configuracion futura; confirmar estado real antes de prometerlo.
- Sin dominio propio todavia.
- Sin pixeles/ads integrations todavia.
- Sin prometer inventario automatico.
- Sin prometer respuesta automatica por WhatsApp.
- Soporte manual.
